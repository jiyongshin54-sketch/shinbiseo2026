# Step 6: 거래처 관리 (스크린샷 기준 정확 레이아웃)

## 개요
판매회사 입장에서 자사 고객(거래처)을 관리하는 화면.
구매회사도 거래처를 관리할 수 있어야 함.

---

## 레이아웃 (스크린샷 기준)

### 상단: 검색 바
```
거래처 관리    거래처명: [________]  거래처 결제방식: [____▼]  [찾기]
```

### 중단: 좌우 2분할

**좌측 (~40%)**: 거래처 목록 (lightblue 배경)
| 순번 | 상호 | 사업자번호 | 대표 | [관리] | [거래] |
|------|------|-----------|------|--------|--------|

- `[관리]` 클릭 → 우측 상세폼에 해당 거래처 정보 로드
- `[거래]` 클릭 → 거래내역 표시 (거래내역PH)
- max-height: ~210px, 스크롤

**우측 (~60%)**: 거래처 상세조회/등록/수정 (darkseagreen 배경)
```
┌─ 거래처 상세조회/등록/수정 ──────────────────────────────┐
│ 거래처ID: [____]     *상호: [________]    사업자번호: [____] │
│ *대표:    [________]  전화번호: [________]  팩스번호: [____]  │
│ *담당자:  [________]  업태: [____________]  종목: [________]  │
│ 이메일:   [________]  주소: [____________]                   │
│ 홈페이지: [________]  비고: [____________]                   │
│                                                            │
│ ┌── (풍원 전용 필드, pungwonTable) ─────────────────────┐  │
│ │ 일반등급: [3.8급~5.0급▼]  마대등급: [3.8급~5.0급▼]       │  │
│ │ 온라인: [________]                                     │  │
│ │ 결제방식: [일결제/월결제▼]  세금계산서: [발행/미발행▼]      │  │
│ └────────────────────────────────────────────────────┘  │
│                                                            │
│ [초기화]                                      [거래처 추가]  │
└────────────────────────────────────────────────────────────┘
```

- 등급 드롭다운: [공란, 3.8급, 3.9급, 4.0급, 4.1급, 4.2급, 4.3급, 4.4급, 4.5급, 4.6급, 4.7급, 4.8급, 4.9급, 5.0급]
- 결제방식: [공란, 일결제, 월결제]
- 세금계산서: [공란, 발행, 미발행]
- 배경색: 기본정보 영역 palegoldenrod, 등급/결제 영역 별도 테이블
- `[관리]` 클릭 시 폼에 데이터 채워지고 `[거래처 추가]` → `[거래처 수정]`으로 변경
- `[거래처 삭제]` 버튼은 수정 모드에서만 표시 (visibility:hidden → visible)

### 거래내역PH (거래 버튼 클릭 시 표시)
거래내역GV (max-height:410px):
| 발행일자 | 구매회사 | 항목수 | 공급가액 | 부가세 | 합계금액 | 지불방식 | 비고 | [수정/출력] |

## API 설계

### GET /api/customers
- params: seller_id, search_name?, payment?
- 거래처 목록 조회

### POST /api/customers
- 거래처 신규 등록

### PATCH /api/customers/[sellerId]/[customerId]
- 거래처 정보 수정

### DELETE /api/customers/[sellerId]/[customerId]
- 거래처 삭제 (→ customers_deleted 테이블로 이동)

### GET /api/receivables
- params: seller_id
- 일결제/월결제 거래처별 미수금 현황 조회
- 계산: 완료 상태가 아닌 주문의 total_amount 합산

### GET /api/customers/[sellerId]/[customerId]/transactions
- 해당 거래처의 거래내역 (거래명세표) 조회

## 권한
- **판매회사 대표/관리자**: 모든 기능
- **판매회사 직원**: 조회만 (등급/결제방식 등 숨김 가능)
- **구매회사**: 자사 거래처 관리 (별도 로직 필요)

## 풍원 전용 필드
- 일반등급, 마대등급, 온라인, 결제방식, 세금계산서
- JavaScript로 `pungwonTable` 표시/숨김 제어
- 현재 로그인 회사가 풍원(00002)이면 표시, 아니면 숨김
