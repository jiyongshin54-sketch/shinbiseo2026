# Step 5: 기성품 주문 (스크린샷 기준 정확 레이아웃)

## 개요
Products 테이블에 등록된 기성 상품을 주문하는 화면.
판매/구매 회사 모두 주문 가능.

---

## 레이아웃 (스크린샷 기준)

### 상단: 거래 대상 선택 바
```
기성품 주문
구분: [판매▼]    거래처: [________] [Guest(대표)    ▼]
```
- 구분 DDL: [판매, 구매] - 판매회사는 "판매" 선택(자사가 파는 입장), 구매회사는 "구매" 고정
- 거래처: TextBox(자동완성) + DDL(고객 목록)

### 중단: 좌우 2분할

**좌측 (~48%)**: 상품 검색 + 후보 목록

검색 입력:
```
┌─────────────────────────────────────────────────────┐
│ 사이즈(예, 15*20)를 입력 하시고 엔터키를 누르세요!!      │ [조회]
└─────────────────────────────────────────────────────┘
```
- 배경: yellow
- placeholder 텍스트 가이드
- 엔터 또는 [조회] 클릭으로 검색

상품 후보 GridView (기성주문후보GV, max-height:612px):
| Category | 품명 | 색깔 | 두께 | 사이즈 | 마대수량 | 일반가 | 마대가 | 수량 |

- 수량 칸: TextBox + "00" 고정 표시 (100장 단위)
  - 예: 사용자가 "2" 입력 → "200"장으로 처리
- 수량 입력 후 엔터 → 자동으로 장바구니에 추가
- 일반가/마대가: 해당 고객의 Level1/Level2 등급에 따른 단가 표시

**우측**: 장바구니 (background: skyblue)

장바구니 GridView (기성CartGV):
| 순번 | 품명 | 색깔 | 두께 | 사이즈 | 단가 | 수량 | 금액 | 묶음 | [빼기] |

- **단가**: 편집 가능 TextBox (판매회사가 가격 직접 수정 가능)
- **묶음**: DDL [공란, 1, 2, 3, 4, 5] - 묶음 수량
- **[빼기]**: 장바구니에서 제거

장바구니 하단 합계 영역:
```
┌─────────────────────────────────────────────────────────┐
│ 거래일자: [2026.03.19]          총계:            11,480  │  (skyblue 배경)
│ 금액합계:              11,480  금액조정: [+▼] [0     ]  │  (aliceblue 배경)
│ 최종공급가:            11,480  부가세:   [▼]  [0     ]  │  (aliceblue 배경)
│ 비고: [____________________________________________]    │
│                                                         │
│      [주문▼] 상태로        [주문 등록]                    │  (MintCream 배경)
└─────────────────────────────────────────────────────────┘
```

- 거래일자: TextBox (판매회사만 수정 가능, 구매회사는 readonly)
- 금액조정: [+/-] DDL + 금액 TextBox
- 부가세: DDL (0% / 10% 등) + 금액 TextBox
- 상태 DDL: [주문, 견적, 준비됨, 수령, 완료] - 판매회사 전용
- `[주문 등록]` / `[주문 수정]` 버튼

### 하단: 안내문 (orange 배경)
```
┌─ 기성품 주문 방법 안내 ────────────────────────────────────┐
│ * 기성품 상품 추가는 키보드의 숫자키와 엔터키만으로도 처리가  │
│   가능합니다 *                                            │
│ 1. 노란색 칸에 size를(예, 15*20) 입력 하고 엔터           │
│ 2. 탭키로 상품 선택 후 수량을 100장 단위로 입력하고 엔터    │
│ 3. 엔터를 치면 자동으로 장바구니에 추가됩니다               │
└──────────────────────────────────────────────────────────┘
```

## 가격 산정 로직

### Level → UnitPrice 매핑
Customers 테이블의 Level1(일반등급), Level2(마대등급)으로 Products 테이블의 단가 결정:

| 등급 | UnitPrice 인덱스 |
|------|-----------------|
| 3.8급 | UnitPrice01 |
| 3.9급 | UnitPrice02 |
| 4.0급 | UnitPrice03 |
| ... | ... |
| 5.0급 | UnitPrice13 |

- 일반가 = Products.UnitPrice[Level1 인덱스]
- 마대가 = Products.UnitPrice[Level2 인덱스]
- 수량 < 마대수량 → 일반가 적용
- 수량 >= 마대수량 → 마대가 적용

### 수량 처리
- 사용자 입력 "2" + "00" = 200장
- 실제 수량 = 입력값 × 100

## 주문 수정 모드
- 주문 관리에서 `[수정]` 클릭 시 기성품 주문 화면으로 이동
- 기존 주문 데이터를 장바구니에 로드
- `[주문 등록]` → `[주문 수정]`으로 버튼 변경
- 주문번호 Label에 기존 OrderID 표시

## API

### GET /api/products
- params: seller_id, search (사이즈/품명 검색)
- 상품 목록 + 고객 등급별 단가 계산

### POST /api/orders
- 새 주문 생성 (orders_m + orders_d INSERT)

### PATCH /api/orders/[orderId]
- 기존 주문 수정
