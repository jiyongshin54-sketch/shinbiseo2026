# Step 7: 주문 관리 (스크린샷 기준 정확 레이아웃)

## 개요
기간별/상태별 주문 검색 및 일괄 처리 화면.
전광판이 "오늘의 실시간 현황"이라면, 주문 관리는 "과거 이력 검색/관리" 역할.

---

## 레이아웃 (스크린샷 기준)

### 검색 필터 (2행, background:silver)

**1행:**
```
조회구분: [판매 내역▼]  보는항목: [주문별▼]  기준일자: [2026.03.01] 📅 ~ [2026.03.19] 📅
[어제] [오늘] [지난달] [이번달]  거래처: [____________] [고객을 선택하세요..▼]
                                                                    [검색]
```

**2행:**
```
기성여부: [____▼]  결제수단: [____▼]  주문상태: [____▼]  거래처 결제방식: [____▼]
```

### 드롭다운 옵션

**조회구분**: [판매 내역, 구매 내역]
- 판매회사: 기본 "판매 내역"
- 구매회사: 기본 "구매 내역"

**보는항목**: [주문별, 세부항목별]
- 주문별: OrdersM 기준 (주문 단위)
- 세부항목별: OrdersD 기준 (품목 단위)

**기준일자**: 시작일 ~ 종료일 (Calendar 팝업)
- 빠른선택: [어제] [오늘] [지난달] [이번달]

**기성여부**: [공란(전체), 기성, 맞춤]

**결제수단**: [공란(전체), 현금, 수금, 카드, 이체]

**주문상태**: [공란(전체), 전체, 집계, 결산, 주문, 준비됨, 수령, 완료, 취소]
- "집계" = 주문+준비됨+수령 (미완료 건)
- "결산" = 완료+취소 (마감된 건)

**거래처 결제방식**: [공란(전체), 일결제, 월결제]

**거래처 DDL**: 자동완성 (TextBox + DropDownList 연동)

---

### 검색 결과 GridView

#### 판매 내역 - 주문별 (검색결과주문판매GV)
| 주문 일자 | 상태 | 구매처 | 등록자 | 기성여부 | 대표물건 | 건수 | 공급가 | 부가세 | 총계 | 지불방식 | 준비 | 수령 | 완료 | 비고 | [수정] |

- "준비" 컬럼: 준비 처리한 사람 이름
- "수령" 컬럼: 수령 확인한 사람 이름
- "완료" 컬럼: 완료 처리한 사람 이름
- `[수정]` 버튼: 해당 주문을 기성품/맞춤품 주문 화면으로 로드 (주문 수정 모드)

#### 구매 내역 - 주문별 (검색결과주문구매GV)
- 판매 내역과 동일하되 "구매처" → "판매처", `[수정]` 버튼 없음

#### 판매 내역 - 세부항목별 (검색결과상세판매GV)
| 주문일자 | 상태 | 구매처 | 등록자 | 종류 | 품명 | 인쇄명 | 도수 | 도안 | 색깔 | 두께 | 사이즈 | 단가 | 수량 | 금액 | VAT | 세부상태 |

#### 구매 내역 - 세부항목별 (검색결과상세구매GV)
- 동일 구조, "구매처" → "판매처"

---

### 하단: 일괄 처리

```
[요약 텍스트: "검색 건수: N건, 합계 공급가: xxx원, 부가세: xxx원, 총계: xxx원"]
결제금액: [________]  결제수단: [현금으로▼/이체로▼/수금으로▼/카드로▼]  [일괄확인]
```

- `[일괄확인]`: 선택된 주문들의 결제를 일괄 처리
- 주문상태가 "수령"인 건들을 "완료"로 일괄 변경

## API 설계

### GET /api/orders/search
```
params:
  seller_id / buyer_id  (조회구분에 따라)
  view_mode: 'order' | 'detail'  (보는항목)
  date_from, date_to   (기준일자)
  ready_made?           (기성여부)
  payment_method?       (결제수단)
  status?               (주문상태, 집계/결산은 서버에서 복합 조건으로 변환)
  customer_payment?     (거래처 결제방식)
  customer_id?          (거래처)
```

### PATCH /api/orders/batch-complete
```
body:
  order_ids: string[]
  payment_method: string
  payment_amount: number
```
- 선택된 주문들을 일괄 완료 처리
