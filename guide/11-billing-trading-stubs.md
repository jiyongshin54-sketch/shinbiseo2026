## 1. 거래명세표 관리 (Main.aspx > 거래명세표PH)

### 검색 영역 (background:silver)
```
거래처: [________] [거래처DDL▼]  기준일자: [시작일] 📅 ~ [종료일] 📅  [검색]
```

### 좌우 2분할

**좌측 (50%)**: 거래명세표 목록
거래명세표GV (max-height:410px):
| 발행일자 | 구매회사 | 항목수 | 공급가액 | 부가세 | 합계금액 | 지불방식 | 비고 | [수정/출력] | [계산서] |

- `[수정/출력]` → 우측 수정 폼 로드 + eTaxBill.aspx 팝업
- `[계산서]` → 세금계산서 발행 연동

**우측**: 거래명세표 등록/수정

헤더 (skyblue 배경):
```
거래일: [________]   거래처: [________] [거래처DDL▼]
```

거래명세표CartGV:
| No. | 카테고리 | 품목 | 규격 | 단가 | 수량 | 공급가액 | 세액 | [수정] | [빼기] |

상세 입력 폼 (blanchedalmond 배경, darkorange 테두리):
```
순번: [__](읽기전용)  카테고리: [DDL▼]
품명: [________]     규격: [________]
단가: [________]     수량: [________]
공급가: [________]   세액: [DDL▼]
[초기화]  [추가]
```

합계 영역 (aliceblue 배경):
```
건수: N    공급가: xxx    부가세: xxx    총계: xxx
지불방식: [DDL▼]    비고: [________]
[거래명세표 등록] (BackColor:MintCream)
```

## API 설계

### 거래명세표
- **GET /api/trading-stubs** - 목록 조회 (seller_id, date_from, date_to, customer_id)
- **POST /api/trading-stubs** - 새 거래명세표 등록
- **PATCH /api/trading-stubs/[tsId]** - 수정
- **GET /api/trading-stubs/[tsId]/details** - 상세 항목 조회

### 청구서
- **GET /api/billing** - 거래처별 청구서 데이터 (seller_id, customer_id, period)

### 세금계산서
- 기존 외부 서비스 연동 (팝빌 등) - 추후 결정
